## Valid/Ready 握手协议详解

这是一种广泛用于数字电路（如 AXI、ICB 总线）的**流量控制握手协议**，用于在发送方（Source）和接收方（Sink）之间可靠地传输数据。

---

### 基本原理

```
发送方(Source)                接收方(Sink)
    │                              │
    │──── valid ──────────────────>│   // 我有数据，数据已就绪
    │<─── ready ───────────────────│   // 我准备好接收了
    │──── data  ──────────────────>│   // 同时传输的数据/地址等
```

**握手成立条件：`valid & ready` 同时为高时，数据在当拍被传输（transaction）。**

---

### 四种组合状态

| valid | ready | 结果 |
|-------|-------|------|
| 0 | 0 | 无数据、无接收，等待 |
| 1 | 0 | 发送方有数据，但接收方忙，数据**必须保持** |
| 0 | 1 | 接收方空闲，等待数据 |
| **1** | **1** | **握手成立，数据传输！** |

---

### 关键规则

**规则1：valid 一旦拉高，在握手成立之前不能撤销**

```
         ┌───────────────────┐
valid    │                   │ (不能在握手前撤销)
─────────┘                   └─────
         ┌───────┐
ready    │       │
─────────┘       └─────────────────
                 ↑
              本拍握手成立
```

**规则2：ready 可以随时变化，没有 valid 时也可以提前拉高**

```
ready ─────────────────┐   ┌─────
valid ─────────────┐   │   │
                   └───┘   └─────
                       ↑
                   握手成立
```

**规则3：valid 不能依赖 ready（避免组合逻辑死锁）**

```verilog
// ❌ 错误：valid 等待 ready 才拉高 → 死锁！
assign cmd_valid = some_ready;

// ✅ 正确：valid 由自身状态决定
assign cmd_valid = (state == SEND);
```

---

### 在 e203 ICB 协议中的体现

ICB（Internal Chip Bus）分为 **CMD 通道** 和 **RSP 通道**，各自独立握手：

```verilog
// CMD 通道（请求）
input  ifu2itcm_icb_cmd_valid,  // IFU说：我有访问请求
output ifu2itcm_icb_cmd_ready,  // ITCM说：我能接受请求

// RSP 通道（响应）
output ifu2itcm_icb_rsp_valid,  // ITCM说：我的响应数据已就绪
input  ifu2itcm_icb_rsp_ready,  // IFU说：我准备好接收响应了
```

**典型时序：**

```
clk       ──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──
            └─┘  └─┘  └─┘  └─┘  └─┘

cmd_valid ──────────────────────────
cmd_ready ────────┐  ┌───────────      // ready 第2拍才来
                  └──┘
               T1  T2  ← T2拍握手成立，CMD 数据在 T2 被采样

rsp_valid          ────────────┐ ┌──
rsp_ready ─────────────────────┘     // RSP T3握手成立
                              T3 ← 响应数据在 T3 被采样
```

---

### 背压（Backpressure）机制

当接收方处理不过来时，可以拉低 ready 来**暂停**发送方：

```
cmd_valid ─────────────────────────────────
cmd_data  ══A═══════════════B══════════════
cmd_ready ──────┐     ┌─────────────┐  ┌──
                └─────┘             └──┘
握手时刻           ↑T1               ↑T2    ← A在T1传输，B在T2传输
```

A 数据在 ready 拉低期间**全程保持不变**，直到握手成立。

---

### 总结

| 特性 | 说明 |
|------|------|
| 同步时刻 | 时钟上升沿采样 |
| 成立条件 | `valid & ready == 1` |
| valid 约束 | 拉高后不可撤销（直到握手） |
| ready 约束 | 无约束，可随时变化 |
| 流控方向 | 接收方通过 ready 控制速率 |
| 死锁预防 | valid 不能依赖 ready |
